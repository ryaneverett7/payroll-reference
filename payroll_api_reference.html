API Error Handling Guide

A complete reference for all error categories returned by payroll partner APIs — including HTTP codes, meanings, examples, and what action to take.

HTTP Status Code Errors
0
API Timed Out / Transmission Failed
Partner's server wasn't prepared to wait — call dropped
Defined in Dashboard

• An error status was returned by the API (code 0)
• HTTP request transmission failed
• Transmission timed out

API timed out on EN and dropped the call. Partner's server was not prepared to wait long enough for a response.

Partner's server was not prepared to wait this long for them to respond with a record.

Defined in Integration Health dashboard for customer.

Escalate

All

429
Rate Limited by Partner
Too many API calls within a 60-second window
Defined in Dashboard

• An error status was returned by the API (code 429)
• Request has been rate-limited by API

Partner rate-limited EN.

EN is sending too many API calls within a 60-second timeframe and the partner has "cut us off."

Defined in Integration Health dashboard for customer.

1. ADP WFN
2. Paycor

500
Internal Server Error
Partner server cannot fulfill request — misconfigured setting
Research / Retry

An error status was returned by the API (code 500)

Partner server cannot fulfill our request due to an "unexpected" condition — which could be a misconfigured setting in their system.

No specific example provided.

Use Retry button. If failure still persists, research issue → Escalate to Team Lead if needed.

Some partners, like iSolved, might return a 500 for an improperly configured deduction code: "Unexpected error has occurred."

200
Body of Partner Transmission Response Was Empty
API responded to POST call with a completely blank body
Escalate

Body of partner transmission response was empty

Partner's API response to our POST call was empty.

We expect a partner to send anything in response to us updating a record, even "[]". If the response is completely blank, we reflect this. No data quality is affected.

Escalate to Slack channel and @RM.
